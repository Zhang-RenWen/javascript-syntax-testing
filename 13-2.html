13-2 Array.prototype.flatMap()

flatMap() 方法是一個疊代方法。可以看作是 map 和 depth 為 1 的 flat 的組合。


console.time();
const words = [`hello`, `world`];
const r1 = words.map((item) => item.split(``)).flat();
console.log(r1);
console.timeEnd();
// ['h', 'e', 'l', 'l', 'o', 'w', 'o', 'r', 'l', 'd']

console.time();
const r2 = words.flatMap((item) => item.split(``));
console.log(r2);
console.timeEnd();
// ['h', 'e', 'l', 'l', 'o', 'w', 'o', 'r', 'l', 'd']

看起來是不是結果都一樣？但是第二種方法的性能會高一些，畢竟第一種要跑兩次方法，中間迴圈多次。如果後面遇到需要先 map 再 flat 一層的情況，我們就可以優先使用 flatMap因為性能較好。

flatMap 的使用我們也不能局限到上面的場景，看上面方法的定義，只要是返回一個陣列就可以。那麽我們就可以實現對結果的新增、刪除、修改


const words = [`hello`, `world`];
const r3 = words.flatMap((item, index) =>
  (index === 0 ? item.split(``).concat(`,`) : item.split(``)));
console.log(r3);// 給單字的間隔加了一個逗號
// ['h', 'e', 'l', 'l', 'o', ',', 'w', 'o', 'r', 'l', 'd']

const number = [1, 2, 3];
const r4 = number.flatMap((item) => [item, item ** item]);
console.log(r4);
// [1, 1, 2, 4, 3, 27]

替代方法


const arr = [1, 2, 3, 4];

arr.flatMap((x) => [x, x * 2]);
// 等同於
const n = arr.length;
const acc = new Array(n * 2);
for (let i = 0; i < n; i++) {
  const x = arr[i];
  acc[i * 2] = x;
  acc[i * 2 + 1] = x * 2;
}
// [1, 2, 2, 4, 3, 6, 4, 8]

需要注意的是，在這種特定情況下，flatMap 方法的性能比 for 迴圈要慢——因為需要創建臨時陣列並進行垃圾回收，同時返回的陣列不需要經常調整大小。但是，在注重靈活性和可讀性的情況下，flatMap 仍可能是正確的解決方案。

案例演示

比較 map 和 flatMap


const arr1 = [1, 2, 3, 4];

arr1.map((x) => [x * 2]);
// [[2], [4], [6], [8]]

arr1.flatMap((x) => [x * 2]);
// [2, 4, 6, 8]

// 只有一層被壓平
arr1.flatMap((x) => [[x * 2]]);
// [[2], [4], [6], [8]]

flatMap 單字處理

還可以處理單詞，但須注意 flatMap 輸入輸出 length 不同


const arr1 = ["it's Sunny in", "", "California"];

arr1.map((x) => x.split(" "));
// [["it's","Sunny","in"],[""],["California"]]

arr1.flatMap((x) => x.split(" "));
// ["it's","Sunny","in", "", "California"]

在 map() 方法過程中新增和刪除元素

flatMap 方法可以用作在 map 方法中添加和刪除元素（修改元素數量）的方法。換句話說，它允許你將一些項處理為另一些項（通過單獨處理每個輸入項），而不總是一對一的。在這個意義上，它的作用類似於 filter 的相反操作。如果要保留該項，則返回一個包含該項的單元素陣列，如果要添加元素，則返回一個包含多個元素的陣列，如果要刪除該項，則返回一個空陣列。 


// 假設我們想要刪除所有負數，並將奇數拆分成偶數和 1
const a = [5, 4, -3, 20, 17, -33, -4, 18];
//         |\  \  x   |  | \   x   x   |
//        [4,1, 4,   20, 16, 1,       18]

const result = a.flatMap((n) => {
  if (n < 0) {
    return [];
  }
  return n % 2 === 0 ? [n] : [n - 1, 1];
});
console.log(result); // [4, 1, 4, 20, 16, 1, 18]

在稀疏陣列上使用 flatMap()

callbackFn 不會被源陣列中的空槽調用，因為 map() 不會調用，而 flat() 將忽略返回陣列中的空槽。


console.log([1, 2, , 4, 5].flatMap((x) => [x, x * 2])); // [1, 2, 2, 4, 4, 8, 5, 10]
console.log([1, 2, 3, 4].flatMap((x) => [, x * 2])); // [2, 4, 6, 8]

在非陣列物件上調用 flatMap()

flatMap() 方法讀取 this 的 length 屬性，然後訪問每個整數索引。如果回呼函數的返回值不是陣列，則始終直接將其附加到結果陣列的末尾。


const arrayLike = {
  length: 3,
  0: 1,
  1: 2,
  2: 3,
};
console.log(Array.prototype.flatMap.call(arrayLike, (x) => [x, x * 2]));
// [1, 2, 2, 4, 3, 6]

// 回呼函式返回的類陣列物件不會被壓平
console.log(
  Array.prototype.flatMap.call(arrayLike, (x) => ({
    length: 1,
    0: x,
  })),
);
// [ { '0': 1, length: 1 }, { '0': 2, length: 1 }, { '0': 3, length: 1 } ]
