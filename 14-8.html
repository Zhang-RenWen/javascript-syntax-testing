14-8 Nullish coalescing operator (??)

空值合並運算子（??）是一個邏輯運算子，當左側的操作數為 null 或者 undefined 時，返回其右側操作數，否則返回左側操作數。

與邏輯或運算子（||）不同，邏輯運算子會在左側操作數為 falsy 時返回右側操作數。也就是說，如果使用 || 來為某些變數設置默認值，可能會遇到意料之外的行為。比如為假值(例如，'' 或 0 ) 時。見下面的例子。 


const foo = null ?? 'default string';
console.log(foo);
// Expected output: "default string"

const baz = 0 ?? 42;
console.log(baz);
// Expected output: 0

在這個例子中，我們使用 Nullish coalescing operator 為變數提供預設值，保證變數不為 null 或者 undefined。 


const nullValue = null;
const emptyText = ""; // falsy
const someNumber = 42;

const valA = nullValue ?? "default for A";
const valB = emptyText ?? "default for B";
const valC = someNumber ?? 0;

console.log(valA); // "default for A"
console.log(valB); // "" (as the empty string is not null or undefined)
console.log(valC); // 42

為變數提供預設值

以往，如果想為一個變數給定預設值，通常的做法是使用邏輯或運算子(||):


let foo;

// foo is never assigned any value so it is still undefined
const someDummyText = foo || "Hello!";

然而，由於 || 是一個布林邏輯運算子，左側的操作數會被強制轉換成布林值用於求值。任何 falsy（0， ''， NaN， null， undefined）都不會被返回。這導致如果你使用0，''或 NaN 作為有效值，就會出現不可預期的結果。 


const count = 0;
const text = "";

const qty = count || 42;
const message = text || "hi!";
console.log(qty); // 42 and not 0
console.log(message); // "hi!" and not ""

空值合並運算符 Nullish coalescing operator 可以避免這種陷阱，其只在第一個操作數為 null 或 undefined 時（而不是其他falsy）返回第二個操作數： 


const myText = ""; // An empty string (which is also a falsy value)

const notFalsyText = myText || "Hello world";
console.log(notFalsyText); // Hello world

const preservingFalsy = myText ?? "Hi neighborhood";
console.log(preservingFalsy); // '' (as myText is neither undefined nor null)

短路

與 OR 和 AND 邏輯運算符相似，當左表達式不為 null 或 undefined 時，不會對右表達式進行求值。 


function A() {
  console.log("A was called");
  return undefined;
}
function B() {
  console.log("B was called");
  return false;
}
function C() {
  console.log("C was called");
  return "foo";
}

console.log(A() ?? C());
// Logs "A was called" then "C was called" and then "foo"
// as A() returned undefined so both expressions are evaluated

console.log(B() ?? C());
// Logs "B was called" then "false"
// as B() returned false (and not null or undefined), the right
// hand side expression was not evaluated

搭配 || 或者 && 一起使用會出現異常。 


null || undefined ?? "foo"; // raises a SyntaxError
true && undefined ?? "foo"; // raises a SyntaxError

若要搭配 || 或者 && 一起使用，建議使用 () 將表達式包裹起來。如此一來表達式內容會優先執行並回傳結果，接著執行 ?? 判斷。 


(null || undefined) ?? "foo"; // return "foo"

可以搭配 Operators-Optional chaining (?.) 一同使用，如以下範例。先使用 (?.) 判斷 foo 中是否存在指定欄位，有的話執行 toUpperCase() 並回傳結果，沒有的話回傳 ?? 右側的例外處理。 


const foo = { someFooProp: "hi" };

console.log(foo.someFooProp?.toUpperCase() ?? "not available"); // "HI"
console.log(foo.someBarProp?.toUpperCase() ?? "not available"); // "not available"
